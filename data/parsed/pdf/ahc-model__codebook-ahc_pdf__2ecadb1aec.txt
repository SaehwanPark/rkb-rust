Tips on Navigating the Codebook 

The Accountable Health Communities (AHC) Model data file is from the Centers for Medicare & Medicaid Services 
(CMS) Center for Medicare & Medicaid Innovation (CMS Innovation Center) AHC Model test. The CCW team obtained 
the AHC data file from CMS and disseminates it. The AHC data file includes all screening and navigation data submitted 
by 31 bridge organizations between May 2018 and April 2023. Many of the fields in the data file are from the AHC 
Health Related Social Needs Screening tool, which is available at: 

•  Standardized Screening for Health-Related Social Needs in Clinical Settings: The Accountable Health Communities 
Screening Tool - National Academy of Medicine (nam.edu)  
•  The AHC Health-Related Social Needs Screening Tool (cms.gov)  

Variables from the AHC Screening tool begin with the prefix “SR_*” (screening results). 

This document is a detailed codebook that describes each variable  in  the AHC data file.  Because the file has  many 
variables, we have included several ways for users to quickly find the information they need: 

•  A complete listing of all variables in the files, in alphabetical order based on their SAS variable names 
•  Individual  entries  for  each  variable  that  contain  a  short  description  of  the  variable,  the  possible  values  for  the 
variable, and, in many cases, notes that discuss how the variable was constructed and should be used 

We have included hyperlinks throughout the codebook to make it easier for analysts to navigate between the table of 
contents and the detailed entries for the individual variables: 

•  Clicking on any variable name in the Table of Contents will take you to the detailed description for that variable 
•  From the detailed  description  for  any  individual  variable,  clicking  on the  ^Back  to  TOC^  link  after  each  variable 
description will take you back to the Table of Contents

Variable Details 

This section of the codebook contains one entry for each variable in the AHC data file. Each entry contains variable 
details to facilitate understanding and use of the variables. 

AHC_BENE_ID 

LABEL:  AHC Beneficiary Identifier 

DESCRIPTION:  Standardized nine-digit unique AHC beneficiary identifier.  

LONG NAME:   AHC_BENE_ID 

TYPE:  CHAR 

LENGTH:   9 

SOURCE:   AHC data system (derived)  

VALUES:   Alphanumeric character string that contains intelligence (e.g., 123019767 is the 19,767th beneficiary 
in the AHC bridge organization B23). 

The first digit is always 1, then the 2
nd and 3
rd digits are the numeric portion of the bridge organization 
identifier (i.e., the “B” from the bridge organization ID is replaced with a “1”). The remaining 6 digits 
are sequential numbers representing unique AHC beneficiaries. 

COMMENT:  This ID is specific to the AHC Model and does not link to any other CMS data file or system. Most AHC 
beneficiaries (94%) have a MSIS_ID or CCW BENE_ID that may be used to join to other CCW Medicaid 
(T-MSIS) or Medicare data files.

DESCRIPTION:  Each organization participating in the AHC program was assigned an AHC bridge organization

LONG NAME:   AHC_BRDG_ORG_ID 

TYPE:  CHAR 

LENGTH:   3 

SOURCE:   AHC data system 

VALUES:   Alphanumeric character string (e.g., B05) 

COMMENT:  This is an identifier assigned within the AHC data system and cannot be linked to any other data set. 

The AHC bridge organization assigned “B09” voluntarily terminated before the AHC model began. 
Three other bridge organizations voluntarily terminated before the AHC Model ended. Specifically, 
B15 ended participation on 5/1/2019, B21 ended participation on 9/1/2019, and B13 ended 
participation on 11/1/2020.

AHC_TRK_CD 

LABEL:  AHC Assistance or Alignment Track 

DESCRIPTION:  AHC assistance or alignment track. 

LONG NAME:   AHC_TRK_CD 

TYPE:  CHAR 

LENGTH:   1 

SOURCE:   AHC data system 

VALUES:   1 = Assistance 
2 = Alignment 

COMMENT:  Bridge organizations participated in one of two AHC model tracks.  

1.  The assistance track identified and randomly assigned to an intervention or control group all 
navigation-eligible Medicare and Medicaid beneficiaries. They provided navigation services to 
those assigned to the intervention group, which involved development of an individualized action 
plan and guidance and support by navigators to help beneficiaries connect with community 
service providers (CSPs) that could help resolve their health-related social needs (HRSNs).  
2.  The alignment track expanded upon the assistance track intervention by also requiring an advisory 
board. Bridge organizations comprised their advisory boards with beneficiaries, health care 
partners, and community service organizations and charged them with identifying and addressing 
gaps in community services relative to community needs. Due to the community alignment aspect 
of the intervention, the alignment track did not have a control group.

BENE_ID 

LABEL:  Encrypted CCW Beneficiary Identifier  

DESCRIPTION:  Encrypted CCW beneficiary identifier. 

LONG NAME:   BENE_ID 

TYPE:  CHAR  

LENGTH:   15 

SOURCE:   AHC data system/CCW (derived) 

VALUES:   15-character alphanumeric string (Ex. 22222222GDDGjJs) 
Null/missing = not enough identifying information to associate with a CCW BENE_ID 

COMMENT:  The AHC Bridge organizations obtained Medicare beneficiary identifying information (e.g., Medicare ID 
card [i.e. Health Insurance Claim Number or HIC, first and last name, date of birth) that the CMS AHC 
Model used to identify a CCW BENE_ID using an internal “crosswalk” process. A BENE_ID was not 
always obtained. 

The BENE_ID is specific to the CCW and is not applicable to any other identification system or data 
source. 

Within the AHC data file, unique beneficiaries are identified using the AHC_BENE_ID. You would only 
use the BENE_ID or MSIS_ID if you were attempting to link AHC data to other CCW data files.

MSIS_ID 

LABEL:  Encrypted Medicaid State Assigned Beneficiary Unique Identifier 

DESCRIPTION:  A state-assigned unique identification number used to identify a Medicaid/CHIP enrolled beneficiary 
and any claims submitted to the system. Also referred to as the Medicaid Statistical Information 
System Identifier (MSIS_ID). Encrypted Medicaid state-assigned beneficiary unique identifier. 

LONG NAME:   MSIS_ID 

TYPE:  CHAR 

LENGTH:   32 

SOURCE:   AHC data system 
CCW (derived) 

VALUES:   Alphanumeric character string, up to 20 characters 
(Ex. 9Q81866B302C768A539B) 
Null/missing = source value is missing or unknown 

COMMENT:  The AHC Bridge organizations obtained Medicaid beneficiary identifying information (e.g., Medicaid ID 
card, state, zip code, first and last name, date of birth) that the CMS AHC Model used to join to the 
CCW MSIS_ID in the Transformed Medicaid Statistical Information System (T-MSIS) Analytic Files (TAF) 
(or in the Medicaid Analytic eXtract (MAX) files for applicable states for 2015 – which is part of a 3-
year look back period for the AHC Model). An MSIS_ID was not always obtained. 

Within the AHC data file, unique beneficiaries are identified using the AHC_BENE_ID. You would only 
use the BENE_ID or MSIS_ID if you were attempting to link AHC data to other CCW data files. For these 
linkages, MSIS_ID must be combined with STATE_CD to uniquely identify a beneficiary. The reference 
year (MSIS_RFRNC_YR) field identifies the earliest year the MSIS_ID was identified during the AHC 
study period. 

This variable is encrypted in the CCW and may not be joined to any other data sets without CMS 
permission.

COMMENT:  Navigation-eligible beneficiaries in the assistance track were randomly assigned to intervention (70%)

assistance track navigation-eligible beneficiaries have a missing value, likely due to a technical issue

opt in/out flag (SR_NVGTN_OPT_IN_OUT_IND) indicating that they were likely not offered navigation 
services. The other seven percent opted out of navigation services (SR_NVGTN_OPT_IN_OUT_IND =
